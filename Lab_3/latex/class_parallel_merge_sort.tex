\doxysubsection{Detailed Description}
Parallel merge sort backed by std\+::thread. 

The list is recursively split in half. Each half is sorted concurrently on a new thread as long as the current recursion depth is below {\ttfamily max\+Depth}. Once the depth limit is reached (or the sub-\/array is small enough) the work falls through to \doxylink{class_sequential_merge_sort}{Sequential\+Merge\+Sort} to avoid spawning too many threads on tiny inputs.

The effective thread count is at most 2\texorpdfstring{$^\wedge$}{\string^}max\+Depth. With the default {\ttfamily num\+Threads=4} the depth is set to log2(4)=2, so at most 4 threads run concurrently.

Example usage\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{ParallelMergeSort\ sorter(4);\ \ \ \ \ \ \ \ \ \ \textcolor{comment}{//\ up\ to\ 4\ threads}}
\DoxyCodeLine{\textcolor{keyword}{auto}\ result\ =\ sorter.sort(\{5,3,8,1\});}

\end{DoxyCode}
